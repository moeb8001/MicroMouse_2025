Section 01

Executive Summary

What we're doing, why it matters, and what we'll deliver.

This feasibility study commissions an external testing facility to determine the minimum detectable particle size (dmin) and associated imaging performance of a high-sensitivity PTV configuration — specifically an intensified camera (pco.dicam C1) paired with a high-power pulsed laser — when applied to ultra-small particles representative of aerodynamic particle release and spitting events.

The study will systematically image certified monodisperse silica microspheres (Microparticles GmbH) spanning 100 nm to 1.0 µm, creating a comprehensive detection performance curve that maps how system capability degrades as particle size approaches the fundamental scattering limits. Results from this external "Setup A" will later be benchmarked against a local "Setup B" (direct-detection camera, lower-power laser) to quantify the performance premium of intensification and high-power illumination.

Section 01-B

General Logic & Workflow

The big picture — how this study is structured and why each stage exists.

The Core Logic in One Paragraph

We have a high-end PTV system (intensified camera + pulsed laser) and we want to know: how small can it see? To answer this, we feed it particles of known, decreasing size and measure what fraction it detects. The result is a detection efficiency curve ηdet(d) — a plot that starts near 100% for large particles and drops to 0% for very small ones. The size where ηdet crosses our threshold (5%) is dmin — the headline result. Everything else in this study (parameter sweeps, controls, statistics) exists to ensure that curve is accurate, trustworthy, and useful.

Study Structure — Five Stages

0
Stage 0 — Preparation (Weeks Before)

Everything before arriving at the facility

Order particles and verify availability. Prepare suspension dilution recipes. Confirm facility equipment (laser, optics, test volume) via pre-test technical call. Source ultrapure water. Ship or transport all consumables. The goal: arrive at the facility with zero unknowns about materials and equipment.

1
Stage 1 — Setup & Calibration (Day 1 Morning)

Establish the measurement baseline

Warm up all instruments. Align the laser sheet. Capture dark frames (camera noise floor) and blank water frames (residue particle floor). These two reference datasets are the foundation — without them, you cannot distinguish real particles from noise or ghost particles at small sizes. Then validate the entire chain with 1 µm particles (the "easy" case) to confirm everything works before starting the critical small-size campaign.

Logic: You're establishing two baselines — the sensor noise floor and the water contamination floor — then confirming the system works end-to-end with a known-good particle size. If the 1 µm validation fails, you troubleshoot now, not after wasting time on 100 nm attempts.

2
Stage 2 — Ascending Size Campaign (Day 1 Afternoon → Day 2)

The core measurement — smallest to largest

After the 1 µm validation, thoroughly clean the system, then run particle sizes in ascending order: 100 → 150 → 200 → 300 → 500 → 1000 nm. At each size: prepare suspension → load atomizer → stabilize concentration (CPC) → acquire frame pairs → quick quality check → flush system → verify clean → next size.

Logic (ascending order): Large particles deposited on surfaces during a run will re-entrain and contaminate subsequent runs. If you run 1 µm first then 100 nm, stray micron-scale particles create false detections that look real. Running small-first means any contamination from a previous run is smaller than the current target — and thus harder to detect, not easier. This eliminates upward cross-contamination bias.

Logic (flush + verify between sizes): CPC monitors the aerosol concentration decay during flushing. You wait until it drops below 1 #/cm³, then capture verification frames to confirm the imaging system also sees nothing. Only then do you load the next size.

3
Stage 3 — Parameter Sweeps at the Knee (Inline with Stage 2)

Explore the sensitivity landscape at critical sizes

After completing the baseline run at a given size, and before flushing for the next size, sweep camera settings (MCP gain, gate width) and laser energy. This happens inline — at the 2–3 sizes nearest the detection cliff (likely 200–500 nm, determined as you go from the Tier 1 results).

Logic: You already have the particles flowing and the system configured. Changing one parameter at a time (e.g., increasing MCP gain from 800V to 900V to 1000V) and acquiring more frames is fast — 10 min per sweep point. This maps how each parameter affects detection, not just whether the particle is detectable. The result is a lookup table: "for d = 300 nm, use gain = 950V, gate = 150 ns" — directly operational knowledge for future experiments.

4
Stage 4 — Controls & Special Runs (Day 2 Afternoon / Day 3)

Validate assumptions and extract bonus information

Repeatability check (re-run one size to check system drift). Concentration sweep (verify ηdet is independent of seeding density). Mixed-size run (test real-world dynamic range). Static particle imaging (pure detection test, position uncertainty measurement). These runs aren't strictly needed for dmin, but they make the dataset much more trustworthy and scientifically complete.

Logic: Controls answer the meta-question: "Can I trust my primary results?" Without them, a reviewer or client can always argue: "Maybe the detection efficiency changed because concentration drifted" or "maybe the system degraded over time." Controls close those arguments.

Data Flow

Input: Certified monodisperse SiO₂ at 6 known sizes + CPC reference concentrations

Raw data: 16-bit TIFF frame pairs + CPC logs + camera/laser metadata per run

Processing: PTV software detects and pairs particles → count Ndetected per run

Analysis: Ndetected / Nexpected (from CPC + measurement volume) → ηdet per size

Output: ηdet(d) curve with confidence intervals, ηpair(d) curve, SNR(d), FPR, σpos(d), optimal settings table

Deliverable: "This system detects particles down to dmin = XXX nm at ηdet ≥ 5%, with optimal settings [gain, gate, laser]. Compared to Setup B (forthcoming), the intensified system extends detection by YYY nm."

Section 02

Objectives & Research Questions

The specific questions this study must answer — prioritized by value.

Primary Research Questions

Q1
What is dmin at ηdet ≥ 5%?

The headline number: smallest particle the system can reliably detect.

Q2
What is the full shape of ηdet(d)?

Not just the threshold — does detection degrade gradually (sigmoid) or fall off a cliff? The curve shape determines practical usability for polydisperse events.

Q3
How does ηpair(d) compare to ηdet(d)?

Pairing requires detection in BOTH frames. Near dmin, stochastic detection means a particle may appear in frame A but vanish in frame B. This "paired vs. single-frame" gap is critical for real PTV.

Secondary Research Questions

Q4
What is the SNR as a function of particle size?

Peak particle intensity / RMS background noise. This is the fundamental quantity that drives all detection and accuracy metrics. Map it experimentally.

Q5
What is the false positive rate at high-sensitivity settings?

When you crank MCP gain to detect 150 nm particles, how many "ghost particles" does noise generate? Critical for trusting results near the detection limit.

Q6
What is the position uncertainty σx,y(d)?

Even if you detect a 200 nm particle, how accurately can you locate its center? Position error propagates directly to velocity error in PTV. Measured via stationary or known-velocity particles.

Q7
What are the optimal settings per size regime?

MCP gain, gate width, laser energy — what's the best recipe for 1 µm, 500 nm, 200 nm, etc.? Deliver a lookup table the operator can use.

Q8
What is the practical dynamic range in a single image?

In a real spitting event, particles range from ~50 nm to >10 µm simultaneously. Can the system see 200 nm without saturating on 2 µm?

Section 03

Physics Framework

Why small particles are fundamentally hard — the scattering cliff.

The ability to detect a particle optically depends on the amount of light it scatters toward the camera sensor. The governing physics transitions between two regimes depending on the size parameter x = πd/λ:

Mie Regime (x ≳ 1)

Particles comparable to or larger than the wavelength. Scattered intensity scales approximately as I ∝ d². For d = 1 µm at λ = 532 nm, x ≈ 5.9 — firmly Mie.

Iscat ∝ d²
Approximate Mie scaling

Rayleigh Regime (x ≪ 1)

Particles much smaller than the wavelength. Scattered intensity plummets as I ∝ d⁶. For d = 100 nm at λ = 532 nm, x ≈ 0.59 — entering Rayleigh.

Iscat ∝ d⁶
Rayleigh scaling
The Scattering Cliff

Going from 1 µm → 100 nm means a 10× reduction in diameter. In the Mie regime, signal drops ~100×. In the Rayleigh regime, signal drops ~1,000,000×. The actual drop for silica at 532 nm falls somewhere in between (transition zone), but expect 10,000 – 100,000× signal reduction. This is not a gradual decline — it is a cliff. The detection efficiency curve ηdet(d) will have a sharp knee, and locating that knee is the central objective.

Implication for Camera Settings

The enormous signal dynamic range means no single camera setting works for all particle sizes. At settings that detect 100 nm particles (maximum gain, wide gate), the 1 µm particles will massively saturate. Conversely, settings optimal for 1 µm will see nothing at 200 nm. The test matrix must include parameter sweeps at each size point.

Diffraction Limit vs. Detection Limit

These are different concepts that are often confused. The diffraction limit (~λ/2NA ≈ 1–3 µm for typical PTV optics) sets the minimum resolvable feature size — all particles below this appear as the same Airy disk. The detection limit is much smaller: you can detect particles well below the diffraction limit as bright spots, you just can't resolve their shape or distinguish them by size from the image alone. This study characterizes the detection limit, not the resolution limit.

Tier 2 — Parameter Sweeps at Critical Sizes (12–16 runs)

At the 2–3 sizes nearest the detection knee (likely 200, 300, and possibly 500 nm), sweep camera and laser parameters to map the sensitivity landscape:

At the knee-region particle size(s), sweep MCP gain voltage: 750V, 850V, 950V, 1000V. This maps how ηdet improves with gain and at what point noise overwhelms the benefit.

Why it matters: Higher gain amplifies both signal AND noise. There's an optimal gain where SNR peaks — beyond that, false positives rise faster than true detections. Finding this optimum is critical operational knowledge.

Deliverable: ηdet vs. MCP gain curve, false positive rate vs. gain, SNR vs. gain.

Sweep gate width: 50 ns, 100 ns, 200 ns. Wider gates collect more photons but also more background (ambient light, dark current, phosphor afterglow).

Interaction with P46 phosphor: P46 has ~300 ns decay time. A gate shorter than the decay time clips useful signal. A gate longer than decay collects noise after the scattering pulse ends.

If the laser has adjustable pulse energy, sweep at 30%, 60%, 100% of maximum. Scattered intensity is directly proportional to illumination intensity, so doubling laser power doubles SNR. But higher laser power can cause background scatter from optics, windows, and residual particles.

Deliverable: ηdet vs. laser energy, identify whether max power is always best or if background scatter creates diminishing returns.

Section 06

Metrics & KPIs

Six core metrics that fully characterize detection performance.

M1 — Detection Efficiency ηdet

ηdet = NPTV / Nexpected
where Nexpected = CCPC × Vmeas × Nframes × (dt / Δtresidence)

Definition: Fraction of particles physically present in the measurement volume that the PTV software actually detects. The denominator requires careful computation from CPC concentration, measurement volume geometry, and flow velocity.

Target: Report for each particle size with 95% confidence interval. Define dmin as the size where ηdet = 5% (i.e., 1 in 20 particles detected).

Gotcha: The measurement volume is NOT the full field of view — it's the intersection of the light sheet thickness, the depth of field of the camera, and the FOV. This must be carefully defined and calibrated.

M2 — Pairing Efficiency ηpair

ηpair = Npaired / Ndetected,frame1
Fraction of detected particles successfully matched across frame pairs

Why it's separate from ηdet: Near dmin, detection becomes stochastic. A particle at the threshold might scatter enough light to be detected in frame A but not frame B (or vice versa). This asymmetric detection kills pairing even when single-frame detection works. The gap between ηdet and ηpair quantifies this "stochastic pairing loss."

M3 — Signal-to-Noise Ratio

SNR = (Ipeak − Ibg) / σbg
Per-particle metric averaged over all detected particles per run

Report as: mean SNR, median SNR, and SNR distribution (histogram). The distribution shape reveals whether particles are marginally detected (narrow distribution near threshold) or robustly detected (wide distribution well above threshold).

M4 — False Positive Rate (FPR)

FPR = Ndetected,blank / Nframes
Ghost particle detections per frame in blank/dark runs

Measured from CTRL-01 and CTRL-02 runs. Any real particle run's detection count must be corrected by subtracting the expected false positives: Ntrue = Nraw − FPR × Nframes.

M5 — Position Uncertainty σpos

Best measured from CTRL-07 (static particles imaged repeatedly). Standard deviation of detected position over many frames for the same particle. Alternatively, from known-velocity flow: σpos can be extracted from the velocity measurement scatter.

σvel = √2 · σpos / Δt
Velocity uncertainty derived from position uncertainty

M6 — Particle Image Diameter dτ

The apparent size of the particle image on the sensor (in pixels). For particles below the diffraction limit, dτ ≈ constant (the Airy disk). Above the diffraction limit, dτ grows with physical size. Tracking dτ(d) reveals at what particle size the system transitions from "detecting bright spots" to "resolving particle images."

Relevance to accuracy: Sub-pixel localization algorithms (Gaussian fitting) work best when dτ ≈ 2–4 pixels. If dτ < 1.5 pixels ("peak-locking"), position accuracy degrades.

Section 07

Test Volume Design

The physical environment where particles are imaged — possibly the most important design decision.

Requirements

The test volume must satisfy competing demands: optical access for the laser sheet and camera, controlled aerosol delivery, minimal background scatter, easy cleaning between runs, and compatibility with the CPC for reference concentration measurement.

Option Analysis

Laminar Flow Cell

Description: Rectangular cross-section channel (e.g., 15×15 mm internal) with optical-quality glass windows on two sides (laser entry + camera view). Aerosol enters from one end, exits to the CPC at the other. Clean air sheaths along the walls prevent particle deposition on windows.

Advantages:

• Known, controlled velocity profile → enables PTV velocity validation
• Continuous particle supply → unlimited acquisition time
• CPC samples the same aerosol → direct concentration reference
• Flow continuously flushes residual particles → fast transitions between sizes
• Sheath flow keeps windows clean

Disadvantages:

• Requires fabrication or availability at facility
• Window reflections can create background scatter
• Flow velocity adds a displacement between frames (must match dt_pair to flow speed)

Recommended dimensions: 15 mm × 15 mm cross-section, ~100 mm length. Anti-reflection coated windows (532 nm AR coat). Flow speed: 0.1–1.0 m/s (adjustable via flow controller).

Sealed Settling Chamber

Description: A closed box (~100×100×100 mm) with optical windows. Fill with aerosol, seal, and image while particles are quasi-stationary (slowly settling or undergoing Brownian motion).

Advantages: Near-zero velocity → large dt_pair acceptable → maximum paired-frame signal collection. Simple construction.

Disadvantages:

• Concentration decays over time (wall losses, settling, diffusion) — especially fast for small particles
• No CPC reference during imaging (CPC requires flow)
• Extremely slow cleanup between sizes (no flushing — must wait or manually clean)
• Brownian motion dominates for small particles: RMS displacement = √(2Dt) where D ∝ 1/d. A 100 nm particle moves ~7 µm/s in RMS, confounding position uncertainty

Open Jet / Free Stream

Description: Aerosol exits a nozzle into open air, laser sheet crosses the jet.

Advantages: No windows (eliminates reflection artifacts). Easy to set up.

Disadvantages:

• Ambient contamination (lab dust, background aerosol) overwhelms signal at small sizes
• Turbulent mixing → chaotic velocity field → pairing becomes difficult
• Concentration hard to control or reference
• Not recommended for this application

Recommendation: Laminar Flow Cell

The flow cell provides the best combination of controlled conditions, continuous CPC reference, fast cleanup, and velocity validation. If the external facility does not have a suitable cell, it can be constructed from optical-quality glass slides and 3D-printed or machined spacers. Anti-reflection coating on the windows is strongly recommended to minimize laser scatter background. Confirm with the facility what test volume they intend to use.

Diffusion Losses for Small Particles

Nanoparticles (100–200 nm) have significant diffusion coefficients. They deposit on tubing walls en route from the atomizer to the test volume. The TSI 3073 output tubing should be as short as possible (< 30 cm) and conductive (metal or conductive silicone) to minimize electrostatic losses. Expect 10–30% losses in tubing for 100 nm particles. This doesn't affect the test (CPC measures at the output), but it means you need higher atomizer output concentrations to achieve target imaging concentrations.

SOP: Preparation

Day Before Testing — Suspension Preparation

The Physics: Why Concentration Matters — Poisson Droplet Loading

The Problem in Plain English

The TSI 3073 atomizer turns your liquid suspension into millions of tiny water droplets. Each droplet is a tiny "sampling cup" — it scoops up whatever particles were floating in that volume of liquid. After the water evaporates, each droplet leaves behind:

• 0 particles → empty droplet, harmless (most common at low concentrations)
• 1 particle → a singlet — this is what we want
• 2+ particles → a multiplet — the particles fuse into a cluster that PTV sees as one bigger particle — this ruins your measurement

μ (mu) is simply the average number of SiO₂ particles per droplet. It's the single number that controls whether your aerosol is mostly singlets (good) or mostly multiplets (bad).

The Goal — and What Determines the Dilution

We want μ ≤ 0.1 for every particle size. At μ = 0.1: ~90% of droplets are empty, ~9.5% contain exactly 1 particle (singlet), and only ~0.5% contain 2+ (multiplet). The singlet ratio R ≈ 95%.

The dilution for each particle size is whatever dilution achieves μ ≤ 0.1. That's it. Everything below is just computing what concentration gives μ = 0.1 for each particle size.

What Determines μ?

Three things:

① How many particles per mL of liquid? → the number concentration Cn
② How big is each droplet? → droplet volume Vd = (π/6)·dd³
③ μ = Cn × Vd → particles per droplet = (concentration) × (volume per droplet)

That's it. More particles in the liquid or bigger droplets → higher μ → more multiplets.

Converting Mass Concentration to Number Concentration

We know our stock is 5 wt% (= 0.05 g/mL). But μ needs the number of particles per mL. To convert, we need the mass of one sphere:

m1 sphere = ρp × (π/6) × dp³ = 1.85 × 0.5236 × dp³
Mass of one SiO₂ sphere. ρp = 1.85 g/cm³ (confirmed from Microparticles GmbH)
Cn = Cm / m1 sphere
Number concentration = mass concentration ÷ mass per particle

Then: μ = Cn × Vd. After algebra (the π/6 cancels):

μ = (Cm / ρp) × (dd / dp)³

⚠ Agglomeration Is Your #1 Enemy

SiO₂ nanoparticles agglomerate via van der Waals forces, especially at smaller sizes. A "100 nm run" where particles have clumped into 300 nm aggregates invalidates your data completely — and you'd never know from the PTV images alone (all sub-diffraction particles look the same). Every step below is designed to minimize and detect agglomeration.

Gather Materials
Required:
— 6× Microparticles GmbH SiO₂ stock bottles (15 mL each, 5% w/v): 100, 150, 200, 300, 500, 1000 nm
— Type 1 ultrapure water (>18 MΩ·cm, <1 ppb TOC) — at least 1 L total
— 6× clean 100 mL glass bottles (DIN thread, compatible with TSI 3073 reservoir)
— 6× intermediate vials (15–50 mL, glass or polypropylene, for serial dilution)

1b
Pre-Sonicate Stock Bottles
Before opening, place each 15 mL stock bottle in the ultrasonic bath for 15 minutes. The stock (5% w/v) may have developed agglomerates during storage and shipping. This step re-disperses the stock before you pipette from it.

Technique: Hold the vial above the bath floor using a clamp or rack — do not let it rest on the bottom. Fill the bath to the manufacturer's recommended level. Suspension Handling Rules During Testing

The 90-Minute Rule

Once a working suspension is loaded into the TSI 3073 and atomization begins, the suspension is usable for approximately 90 minutes before evaporative concentration drift and re-agglomeration make it unreliable. If a run + parameter sweeps at one particle size exceed 90 minutes, prepare and load a fresh bottle of the same working suspension.

Between-Size Atomizer Cleaning

When switching particle sizes, the atomizer reservoir and internal tubing must be cleaned to prevent cross-contamination:

① Remove the used suspension bottle.
② Rinse the reservoir bottle 3× with ultrapure water (fill halfway, swirl vigorously, discard).
③ Run the atomizer with a fresh bottle of ultrapure water for 2 minutes to flush internal wetted surfaces.
④ Discard the rinse water and load the next size's working suspension.
⑤ Allow 3–5 min for the CPC to stabilize before beginning data acquisition.

QUESTION
Is concentration drifting up or down?
DRIFTING DOWN
Likely causes: suspension settling in atomizer (re-shake/re-sonicate), atomizer reservoir running low (refill), tubing clog developing. If drift > 30% during a run, discard data and re-acquire after stabilizing.
DRIFTING UP
Likely causes: contamination source (check lab doors, HVAC), agglomerates breaking apart in atomizer (unusual but possible). Wait for stabilization; if persistent, check filtered air supply for leaks.

IN DARK FRAMES → Camera/sensor noise

IN BLANK WATER ONLY → Residue particles

Section 13

Data Quality Gates

Accept/reject criteria for each data run.

Gate 1: File Format

All image frames MUST be saved as 16-bit TIFF. Any frames saved as 8-bit or JPEG are automatically rejected — the reduced dynamic range destroys the ability to distinguish dim particles from noise.

Gate 2: Minimum Detected Count

For runs where ηdet > 0: at least 200 particles must be detected to achieve ±7% uncertainty on ηdet at 95% confidence (from binomial statistics). If fewer than 200 detected after the planned frame count, extend acquisition.

Nmin = (zα/2)² · η(1−η) / E² ≈ 200 for E = 0.07, η ≈ 0.1
Binomial confidence interval requirement

Gate 3: CPC Stability

CPC concentration must not drift more than ±30% during any single run. If it does, segment the data into stable sub-periods and analyze each independently, or reject and re-run after re-stabilizing.

Gate 4: Background Contamination

Pre-run verification frames (step C4) must show ≤ 2 particles in 50 frames before starting a new particle size. Otherwise, continue flushing.

Gate 5: No Saturation

For runs with large particles (≥ 500 nm) at high sensitivity settings: check that peak pixel values do not hit the 16-bit maximum (65,535 counts). Saturated particles have unmeasurable intensity and their positions are poorly determined. If saturated, reduce gain or laser energy and re-acquire.

Gate 6: Metadata Completeness

Every run's data package must include: frame images (TIFF), camera settings log, CPC concentration log with timestamps, laser settings, room temperature, run notes. Incomplete packages are flagged for follow-up before leaving the facility.

Section 16

Value-Add Tests

Additional tests integrated into the campaign — approved, low-effort, high-value.

These tests are woven into the main campaign schedule. Most cost 10–30 minutes of extra acquisition time and provide data that would be impossible to obtain after leaving the facility.

VA-1: SMPS Verification Scans

When: Before each particle-size run, immediately after CPC concentration stabilizes.
What: Run a 3-minute SMPS scan of the aerosol output at the working concentration. Record the full mobility-size distribution.
Why: Confirms singlets (sharp peak at nominal dp), detects doublets (secondary peak at ~1.26× dp), and verifies no unexpected sizes. This is your Poisson framework quality check — if you see a doublet peak, your concentration is too high.
Duration: ~5 min per size (3 min scan + 2 min setup). Total: 30 min across 6 sizes.
Equipment: SMPS (to be arranged). Connect to the same aerosol line feeding the test volume.

VA-2: Single-Frame Acquisition

When: After each double-frame PTV acquisition, at the same particle size and settings.
What: Switch pco.dicam C1 to single-image mode (readout noise drops from 2.2 to 1.1 e⁻). Acquire 500 single frames at identical gain/gate/laser.
Why: Separates detection from pairing. Near dmin, a particle might be detected in frame A but not frame B (stochastic detection). Single-frame mode gives a purer ηdet measurement without the pairing penalty. Comparing ηdet,single to ηpair,double directly quantifies the "stochastic pairing loss" — this gap is unmeasured in the literature and is paper-worthy on its own.
Duration: ~10 min per size to switch mode and acquire. Total: ~1 hr across 6 sizes.
Key insight: If ηdet,single(200nm) = 30% but ηpair,double(200nm) = 9%, the pairing gap reveals that Ppair ≈ Pdetect² — a quadratic penalty nobody has measured.

VA-3: Aperture Sweep

When: At 1–2 sizes near the detection knee (e.g., 200 nm and 300 nm), during the Tier 2 parameter sweep phase.
What: Turn the aperture ring on the imaging lens: f/2.8 → f/4 → f/5.6 → f/8. Acquire 500 frame pairs at each f-stop. Keep all other settings constant.
Why: Wider aperture = more light collected = better SNR, but shallower depth of field = thinner measurement volume. This maps the aperture–SNR–DOF trade-off experimentally. Trivial to do (turn a ring), operationally valuable for anyone designing a PTV system for small particles.
Duration: ~10 min per f-stop × 4 stops × 2 sizes = ~80 min. Can be done within the Tier 2 time budget.

VA-4: Polydisperse Mixed-Size Run (Expanded)

When: Day 2 afternoon or Day 3, after all monodisperse runs are complete.
What: Prepare 3 mixed suspensions and run each:
• Mix A — Easy: 500 nm + 1000 nm (both easily detectable, tests dynamic range)
• Mix B — Challenging: 200 nm + 500 nm + 1000 nm (tests whether bright 1 µm particles create halos that mask dim 200 nm particles)
• Mix C — Extreme: 100 nm + 300 nm + 1000 nm (simulates a real spitting event with wide size spread)
How to prepare mixes: Use the working suspensions already prepared. Mix equal volumes of each size's working suspension into a fresh 100 mL TSI bottle. Since each size has a different concentration (optimized for singlets), the mix will have a multi-modal size distribution — which is exactly what a real event looks like.
Why: Directly tests the real-world scenario. Key questions: can PTV detect 200 nm when 1 µm particles are also present? Do bright particles bloom and saturate the sensor locally? Does the pairing algorithm preferentially match bright particles?
Duration: ~20 min per mix × 3 mixes = ~1 hr. Run these LAST (after monodisperse campaign) because the mixed particles contaminate the system.

VA-5: Static Particle Slide Imaging

When: Day 2 afternoon or Day 3.
What: Prepare a glass slide with deposited particles: pipette 1 µL of dilute (0.001%) suspension of 300 nm and 1000 nm particles onto separate spots on a clean glass slide. Let dry. Place slide in the laser sheet plane, image stationary particles with 200+ frames.
Why: Eliminates velocity, pairing, and concentration as variables — pure detection test. By imaging the same particle 200× times, you get position uncertainty σpos directly from the scatter in detected positions. You also get per-particle SNR statistics and an intensity-vs-size calibration.
Equipment needed: Clean glass microscope slides, micropipette.
Duration: ~30 min (slide prep + imaging).

VA-6: Gate Width Sweep

When: At 1–2 sizes near the detection limit (e.g., 200 nm), during Tier 2 sweeps.
What: Sweep the pco.dicam C1 gate width: 20 ns, 50 ns, 100 ns, 200 ns, 500 ns. Acquire 500 frame pairs at each. Same gain and laser settings.
Why: Wider gate = more signal photons collected (longer integration) but also more noise (dark current, phosphor afterglow, background). There's an optimal gate width that maximizes SNR. For P46 phosphor (decay to 10% in 0.2–0.4 µs), gates beyond ~500 ns collect mostly afterglow noise. Finding the optimum experimentally gives operators a direct recommendation.
Duration: ~15 min per size (software setting change, fast).

VA-7: MCP Dark Count Mapping

When: Day 1 setup, after warm-up, before any particle runs.
What: With NO laser and NO particles (lens cap on), acquire 200 frames at each MCP gain voltage: 700V, 750V, 800V, 850V, 900V, 950V, 1000V (or whatever the gain range is).
Why: This maps the camera's intrinsic noise floor as a function of gain. The MCP amplifies both signal AND dark counts. At some gain level, dark count "spikes" become indistinguishable from dim particle images — that's the noise-limited ceiling. This data is essential for the stochastic detection model and for setting realistic gain limits.
What "dark counts" look like: Random bright spots on an otherwise dark image. They're caused by thermal electron emission from the photocathode, which the MCP amplifies. At low gain, they're rare and dim. At high gain, they become frequent and bright — creating "ghost particles" that trigger the PTV detector. The rate depends on photocathode type: S20 has ~1,500 dark counts/s/cm², GaAsP has ~10,000/s/cm², GaAs has ~30,000/s/cm² (from the pco.dicam C1 datasheet).
Duration: ~10 min total.

VA-8: Laser-Only + Water-Only Background Runs

When: Day 1 setup, right after dark count mapping (VA-7).
What: Two separate runs at the highest sensitivity settings you plan to use:
• Laser-only: Laser firing into the test volume with NO aerosol (filtered air only). 200 frame pairs. This captures scatter from windows, optics, and surfaces.
• Water-only: Atomize pure ultrapure water (no particles) through the diffusion dryer and into the test volume. Laser on, camera at max sensitivity. 500 frame pairs. This captures residue ghost particles from dissolved impurities in the water.
Why: Combined with VA-7, these three runs give you the complete noise budget: noise = dark counts + laser background + residue particles. Any detection at small particle sizes must exceed ALL three noise sources to be real. The water-only run also validates your ultrapure water quality — if you see > 50 "particles" per frame, your water quality is insufficient.
Duration: ~20 min total for both runs.

Post-Processing: Computing ηdet and ηpair

Step-by-step walkthrough of how to turn raw frame data into detection and pairing efficiency numbers.

Detection Efficiency ηdet

The formula is simple. The challenge is getting Nexpected right.

ηdet = Ndetected / Nexpected
Fraction of particles physically present that the PTV software actually detects

Getting Ndetected

This is the easy part. Run PTV detection software on your frames. It outputs a list of detected particle positions per frame. Count them. Sum over all frames in the run: Ndetected = total detected particles.

Getting Nexpected — The Hard Part

Nexpected = "how many particles were actually in the measurement volume during each frame." You cannot measure this directly — you compute it from CPC concentration and measurement volume:

Nexpected per frame = CCPC × Vmeas
CPC concentration (#/cm³) × measurement volume (cm³)

where Vmeas = FOV width × FOV height × laser sheet thickness (or depth of field, whichever is smaller).

The Calibration Trick: Use 1 µm to Find Vmeas

Vmeas depends on laser sheet thickness, depth of field, and FOV — all of which require careful measurement. But there's a shortcut: use the 1 µm validation run to back-calculate Vmeas.

At 1 µm, the system should detect a large fraction of particles (assume ηdet(1µm) is high). If PTV detects Ndet particles per frame and the CPC reads CCPC, then:

Vmeas = Ndet,1µm / (ηdet,assumed × CCPC,1µm)
Back-calculate Vmeas from the 1 µm reference run

If you assume ηdet(1µm) ≈ 1.0 (upper bound → gives minimum Vmeas), or use ηdet(1µm) from literature (~0.5–0.9 for well-optimized systems), you get Vmeas. Then use this SAME Vmeas for all other sizes. The systematic error from the ηdet(1µm) assumption cancels when you compute ηdet ratios between sizes.

Correction: Diffusion Losses in Tubing

The CPC samples downstream of the test volume. Small particles (100–200 nm) experience diffusion losses in the tubing — some stick to tube walls before reaching the CPC. This means CCPC understates the true concentration in the test volume, making ηdet appear artificially high. Correct with the Gormley-Kennedy equation: penetration P = exp(−ξ) where ξ depends on tube length, diameter, flow rate, and particle diffusivity D = kT/(3πμdp). For 100 nm particles in 30 cm of 8 mm tubing at 1 L/min: ~10–20% loss. Worth computing but not a showstopper.

Pairing Efficiency ηpair

This is simpler — no CPC or volume needed:

ηpair = Npaired / Ndetected,frame1
Fraction of detected particles successfully matched across frame pairs

Ndetected,frame1 = particles detected in the first frame of each pair. Npaired = particles successfully matched to a partner in the second frame by the PTV tracking algorithm. Both numbers come directly from the PTV software output — no calibration needed.

Confidence Intervals

Both ηdet and ηpair are proportions (successes / trials). The 95% confidence interval:

CI = η ± 1.96 × √(η(1−η) / N)
Binomial confidence interval for proportion η from N trials

Example: At ηdet = 10% with N = 200 detected particles: CI = 10% ± 4.2%. At N = 1000: CI = 10% ± 1.9%. This is why the minimum 200 detection threshold matters.

Worked Example

Run R-04: dp = 300 nm

CPC concentration: C = 2,500 #/cm³
Calibrated measurement volume (from 1 µm run): Vmeas = 0.015 cm³
Frames acquired: 2,000 pairs

Nexpected per frame = 2,500 × 0.015 = 37.5 particles/frame
Nexpected total = 37.5 × 2,000 = 75,000 particles

PTV software detects: 8,400 particles total → ηdet = 8,400 / 75,000 = 11.2%
CI: 11.2% ± 0.7% (N = 8,400 is large → tight CI ✓)

PTV pairs: 2,100 of 8,400 → ηpair = 2,100 / 8,400 = 25.0%
Note: ηpair = 25% ≈ (√ηdet)² is consistent with stochastic detection model (Ppair ≈ Pdet² = 0.33² ≈ 0.11 — hmm, doesn't match. This would be worth investigating!)
